# 1단계 — 견적·계약

당신은 외주 프로젝트의 **견적과 계약 초안**을 생성합니다.
**가장 중요**: 누락 시 외주 책임 직결. 정확도 최우선 (Opus 권장).

## 입력 자료
$ARGUMENTS

## 절차

### Step 1 — 인테이크 로드 + §8 답변 수신 점검 (★ 게이트)
- 인자에 경로가 있으면 그대로 Read
- 없으면 `ls -t ./intake/*.md | head -1`으로 최신 파일 자동 선택
- 인테이크 폼이 없으면 "먼저 `/인테이크` 실행이 필요합니다" 안내 후 종료
- **인테이크 폼의 §8 클라이언트 답변 수신 표를 확인**:
  - 5개 답변 칸이 모두 비어있으면 → 사용자에게: "클라이언트 답변이 아직 안 들어왔습니다. 어떻게 진행할까요?" 묻기 (AskUserQuestion: 답변 기다리기 / 추정치로 진행)
  - 부분 답변(1~4개)이면 → 받은 항목만 사용, 나머지는 업계 평균 추정치 + 위험 가산
  - 5개 모두 채워졌으면 → 정상 진행

### Step 2 — 메모리 인용 로드
다음 메모리 자동 인용:
- `[[feedback-outsource-pricing-strategy]]` — 첫 거래 마진 양보 + 다음 정식 의뢰 우선 협상권
- `[[feedback-outsource-scope-protection]]` — 5조항 동의 + Out-of-Scope 명문화 + 영업 확장 별도 견적

### Step 3 — 견적 산출
다음 항목을 합산:

| 항목 | 계산 |
|---|---|
| 기본 개발비 | 인테이크의 In-Scope 항목 × 단가 (시장가 기준) |
| 복잡도 가산 | 인증·결제·실시간·AI 등 포함 시 +20~50% |
| 토큰비 추정 | 예상 토큰 사용량 × Opus/Sonnet 단가 ÷ 마진율 |
| 리스크 가산 | 위험 매트릭스 점수 × 0~30% |
| 마진 | 첫 거래는 양보 가능 (메모리 인용) |
| **소계** | 위 합산 |
| VAT (10%) | 별도 표기 |
| **총액** | |

### Step 4 — 산출물 생성

**A. 견적서** (`./contracts/견적서-YYYY-MM-DD.md`):
```markdown
# 견적서

작성일: YYYY-MM-DD
프로젝트: {이름}
견적 유효기간: 30일

## 범위 (In-Scope)
- [인테이크 §2-3 기반]

## 범위 외 (Out-of-Scope) — 추가 요청은 별도 견적
- [인테이크 §6 + 5조항 기반]

## 산출물
- 기획문서 (CLAUDE.md)
- 디자인 시안 분해
- 소스코드 + 인계 매뉴얼
- 인수확인서 + 30~90일 하자보수

## 일정
- 마일스톤 N개, 총 X주

## 금액
| 항목 | 금액 |
|---|---|
| ... | ... |

**총액: ₩X,XXX,XXX (VAT 별도)**

## 결제 일정
- 착수금 30% — 계약 체결 시
- 중도금 40% — 검수 통과 시
- 잔금 30% — 인수확인서 서명 시

## 하자보수
- 기간: 30~90일 (협의)
- 무상 범위: 명시된 In-Scope 기능의 버그
- 별도 견적: Out-of-Scope·추가 기능·튜닝

## SLA (서비스 수준 약정) — §15.3
하자보수 기간 동안 적용:
| 항목 | 약정값 |
|---|---|
| 가동률(uptime) | 99.5% (인프라 등급 따라 조정 / Vercel Pro·전용서버는 상향) |
| 장애 1차 응답 | critical 영업일 4h, 일반 영업일 24h |
| 모니터링 | UptimeRobot + Sentry |
- 하자보수 기간 후 운영은 별도 월 정액 계약으로 전환.
- 소규모 건은 "best-effort·무보장"으로 명문화 가능 (그것도 SLA의 일종).
```

**B. 계약서 초안** (`./contracts/계약서-YYYY-MM-DD.md`):
- 견적서 + 다음 5조항 자동 삽입:
  1. In/Out-of-Scope 명문화 (Out-of-Scope 추가 요구는 별도 견적)
  2. 인수 시점 정의 (UAT 통과 + 인수확인서 서명)
  3. 하자보수 범위 (버그=무상, 추가=유상)
  4. 클라이언트 책임 (자료 제공·승인 지연 시 일정 보정)
  5. 영업 확장 시 별도 견적표 적용
- NDA가 필요한 프로젝트는 별도 부속서로 첨부 옵션 안내

**C. 입금 트래커** (`./contracts/입금트래커.md`):
```markdown
# 입금 트래커

| 단계 | 비율 | 금액 | 마감 | 입금일 | 상태 |
|---|---|---|---|---|---|
| 착수금 | 30% | ₩... | 계약일+7 | | ⏳ |
| 중도금 | 40% | ₩... | 검수통과+7 | | ⏳ |
| 잔금 | 30% | ₩... | 인수서명+7 | | ⏳ |
```

### Step 5 — ExitPlanMode 검토
견적서 초안을 사용자에게 보여주고 ExitPlanMode 호출.
사용자가 검토 후 클라이언트에게 전달.

### Step 6 — PDF 변환 안내 (선택)
클라이언트가 마크다운을 못 읽으면 pandoc으로 PDF 변환:
```bash
# 필요 시 한 번만 설치
brew install pandoc basictex
# 변환
pandoc ./contracts/견적서-*.md -o ./contracts/견적서.pdf --pdf-engine=xelatex -V mainfont="Apple SD Gothic Neo"
pandoc ./contracts/계약서-*.md -o ./contracts/계약서.pdf --pdf-engine=xelatex -V mainfont="Apple SD Gothic Neo"
```

### Step 7 — 다음 단계 안내
- "계약서 서명 + 착수금 입금 확인 → `/입금확인 착수금` 실행 → `/기획`으로 진행하세요."
- "착수금 입금 전 2단계 진입은 §11 게이트 ①에 의해 차단됩니다."
