# 0단계 — Discovery (요구사항 인테이크)

당신은 외주 프로젝트의 **첫 요구사항 정리**를 담당합니다.
이 단계의 목표: 자연어 요구사항을 분해해 **§1 견적·계약**의 입력값을 만든다.

## 입력 자료
$ARGUMENTS

## 절차

### Step 1 — 입력 형식 판별
- 파일 경로처럼 보이면 → Read로 읽기
- transcript가 UTF-16 BE 한국 통화녹음앱 출력이면 → `iconv -f UTF-16BE -t UTF-8` 변환 (메모리 `reference-encoding-calltranscript` 참고)
- 자연어 본문이면 → 그대로 사용

### Step 2 — 요구사항 분해
원문에서 다음을 추출해 정리:

1. **명시적 요구** — 클라이언트가 직접 말한 것
2. **암묵적 요구** — 명시 안 됐지만 상식상 필요한 것 (로그인이 필요한 서비스인데 회원가입 언급 없음 등)
3. **비기능 요구사항 후보** — 성능·보안·접근성·디바이스 지원 (명시 안 됐어도 추정)
4. **사업 맥락** — 클라이언트 산업, 사용자 페르소나, 경쟁사
5. **숨은 위험** — 데드라인 압박·예산 제약·외부 의존성

### Step 3 — 모호한 부분 식별 → "다시 물어볼 5가지"
견적·기획 전에 반드시 명확화가 필요한 5개 질문을 자동 생성:
- 일정 (특히 정부지원·런칭 등 외부 데드라인 여부)
- 예산 범위
- 사용자 규모 (1매장? 100매장? 글로벌?)
- 데이터 민감도 (개인정보·결제·의료 등)
- 인수 후 운영 주체 (클라이언트가 직접? 위탁?)

상황별로 5개를 동적으로 조합. AskUserQuestion으로 1-3개씩 묶어 사용자에게 묻기.

### Step 4 — 인테이크 폼 작성

`./intake/YYYY-MM-DD-{프로젝트명}.md`로 저장:

- ...

### Step 5 — 다음 단계 안내
사용자에게: "5가지 질문을 클라이언트에게 보낸 뒤 답이 오면 `/견적` 명령으로 진행하세요. 답이 일부만 와도 `/견적`은 추정치로 초안 생성 가능합니다."
