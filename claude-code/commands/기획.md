# 1단계 — 기획 (Source of Truth 확정)

당신은 외주 프로젝트의 **기획문서**를 작성합니다.
이 문서는 프로젝트 전체에서 절대 흔들리지 않을 기준이 됩니다.

## 클라이언트 원문 요구사항
$ARGUMENTS

## 절차

### Step 0 — §11 게이트 ① 착수금 입금 확인 (★ v2.0)

§2 진입 전 다음을 확인합니다:
- `./contracts/입금트래커.md` 존재 여부 점검
- 존재하면 착수금 행에 ✅ 마킹 있는지 확인:
  ```bash
  grep -E '착수금.*✅' ./contracts/입금트래커.md
  ```
- 결과 분기:
  - ✅ 있음 → 정상 진행 (게이트 통과)
  - ✅ 없음 → 사용자에게 알림: "착수금 입금이 확인되지 않았습니다. 입금이 완료됐다면 `/입금확인 착수금`을 먼저 실행해주세요. 입금 전이라면 §2 진행을 멈춥니다."
  - 트래커 파일 자체가 없음 → "이전 단계 `/견적`이 누락된 것 같습니다. 확인 부탁드립니다."
- 사용자가 명시적으로 "그래도 진행"하면 그 사실을 기록한 뒤 진행 (외주 책임 한계 보호를 위한 추적 가능성).

### Step 0.5 — 사전 점검
- 현재 디렉토리가 빈 프로젝트인지 또는 기존 프로젝트인지 확인
- 이미 `CLAUDE.md`가 있으면 사용자에게 확인:
  - "기존 기획문서가 있습니다. 덮어쓸까요? 추가 작성할까요?"

### Step 1 — 요구사항 분해
원문 요구사항에서 다음을 추출:
- **명시적 요구**: 클라이언트가 직접 언급한 것
- **암묵적 요구**: 명시는 안 됐지만 상식적으로 필요한 것
- **모호한 부분**: 더 명확화가 필요한 것 → 사용자에게 질문

질문이 필요하면 AskUserQuestion으로 한 번에 정리해서 묻기. 1-3개 핵심 질문만.

### Step 2 — 기획문서 작성

`~/.claude/templates/기획안.md` 템플릿을 기반으로 다음 섹션 채워서 작성:

1. **프로젝트 개요**
2. **클라이언트 원문 요구사항** (원문 그대로 보존)
3. **In-Scope (이 프로젝트에 포함)** — 명확히 항목별
4. **Out-of-Scope (이 프로젝트 제외)** — 가장 중요. 추가 요청 거절 근거
5. **기능 명세** — In-Scope 각 항목의 상세
6. **비기능 요구사항** — 성능/보안/접근성/브라우저 지원
7. **기술 스택 결정** + 결정 사유
8. **마일스톤·산출물·일정**
9. **리스크 및 가정**
10. **변경 관리 절차** — Out-of-Scope 항목 추가 시 어떻게 할지

### Step 3 — ExitPlanMode로 승인

작성한 기획안을 사용자에게 보여주고 ExitPlanMode 호출.
사용자 승인 후에만 다음 단계로 진행.

### Step 4 — 확정 및 저장

승인되면:
1. 프로젝트 루트에 `CLAUDE.md`로 저장
2. 동일 내용을 클라이언트 전달용 PDF 변환 가능한 마크다운으로도 `./기획안.md`로 저장
3. git 저장소 아니면 `git init` 실행하고 첫 커밋: "1단계 기획문서 확정"

### Step 5 — 다음 단계 안내

사용자에게 다음 단계 옵션 제시:
- "이제 2단계 디자인으로 넘어갑니다. Claude Design에서 와이어프레임을 받아오세요."
- 또는 "기술 스택 셋업부터 시작할까요?"
