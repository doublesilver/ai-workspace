# 8단계 — 하자보수 (Post-Delivery Monitoring)

당신은 인수 후 **자동 헬스 모니터링**과 **요청 분류**를 담당합니다.
인수확인서 서명 직후 또는 인수 후 클라이언트 요청 발생 시 호출.

## 입력
$ARGUMENTS

## 모드 판별
- 인자가 "30일"/"60일"/"90일" 등 기간이면 → **초기 셋업 모드**
- 인자가 "분류 {내용}"이면 → **요청 분류 모드**
- 인자가 없으면 → 사용자에게 어느 모드인지 묻기

---

## 모드 A — 초기 셋업

### A-1 사전 확인
- `./acceptance/YYYY-MM-DD-인수확인서.md` 존재 확인
- 없으면 "`/인수`를 먼저 진행하세요" 안내 후 종료

### A-2 하자보수 기간 설정
- 인자에서 기간 추출 (기본 30일)
- 시작일·종료일 계산 → `./warranty/시작-YYYY-MM-DD.md` 작성

### A-3 모니터링 도구 셋업
프로젝트 스택 감지 후 권장:

| 스택 | 권장 도구 | 셋업 |
|---|---|---|
| Next.js + Vercel | Vercel Analytics + Sentry | 자동 통합 |
| Railway 백엔드 | Railway logs + Sentry | webhook 설정 |
| Python FastAPI | Sentry SDK + 로그 파일 watch | 코드 추가 |
| 전 스택 공통 | UptimeRobot (무상 50 monitor) | URL 등록 |

각 도구별 셋업 명령·코드 스니펫을 사용자에게 출력.

### A-4 자동 트리아지 webhook (선택)
Sentry/Railway에서 알림을 받아 Claude가 자동 분류하는 흐름:
- Sentry → email/Slack → 사용자 확인 → `/하자보수 분류 {issue 내용}` 호출
- 또는 GitHub Actions cron으로 매일 1회 로그 점검

### A-5 종료 알림 예약
- 종료 7일 전 / 1일 전 자동 알림 (cron 또는 캘린더 ICS 파일 생성)
- 종료일 도래 시 클라이언트에 "하자보수 기간 종료" 안내 메시지 생성

### A-6 산출물
- `./warranty/시작-YYYY-MM-DD.md` — 시작 기록
- `./warranty/모니터링가이드.md` — 도구 셋업 가이드 (인계 매뉴얼 보강)
- `./warranty/종료-YYYY-MM-DD.md` — 종료일 placeholder

---

## 모드 B — 요청 분류

들어온 클라이언트 요청을 다음 기준으로 분류:

### B-1 기획문서·인수확인서 대조
- 프로젝트 루트 `CLAUDE.md` In-Scope·Out-of-Scope 읽기
- `./acceptance/YYYY-MM-DD-인수확인서.md` §3 하자/유상 범위 읽기

### B-2 분류 매트릭스

| 요청 유형 | 판정 | 처리 |
|---|---|---|
| In-Scope 기능의 명백한 버그 | **하자 (무상)** | 즉시 픽스 → 검수 → 배포 |
| In-Scope 기능의 의도된 동작에 대한 불만 | **하자 아님 (조정 요청)** | 사용자에게 보고, 필요 시 별도 견적 |
| Out-of-Scope 명시 항목의 요청 | **추가 (유상)** | `/CR`로 이관, 별도 견적 |
| In/Out 모호 항목 | **사용자 판단 필요** | 양측 근거 정리해서 보고 |
| 클라이언트 인프라 변경에 따른 재작업 | **추가 (유상)** | 인수확인서 §3 인용해 별도 견적 |
| 보안 Critical 패치 | **하자 (무상)** | 즉시 픽스 (기간 무관, 책임 보호 차원) |

### B-3 출력
- 판정 결과 (1줄)
- 근거 (CLAUDE.md / 인수확인서의 어느 조항)
- 권장 다음 액션 (즉시 픽스 / `/CR` 진입 / 클라이언트에 조정 답변)
- 클라이언트 전달용 메시지 초안 (한국어 존댓말)

### B-4 기록
모든 분류 결과는 `./warranty/log.md`에 누적 append:
```markdown
## YYYY-MM-DD HH:MM
- 요청: ...
- 판정: 하자(무상) / 추가(유상) / 사용자 판단
- 근거: ...
- 처리: ...
```
