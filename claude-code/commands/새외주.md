# 새 외주 프로젝트 부트스트랩

프로젝트명: $ARGUMENTS

## 절차

### Step 1 — 디렉토리 생성
- `~/projects/외주/$ARGUMENTS/` 생성 (없으면)
- 그 디렉토리로 이동
- 이미 존재하면 사용자에게 확인 후 진행

### Step 2 — 표준 구조 생성 (v2.0)

```
~/projects/외주/$ARGUMENTS/
├── CLAUDE.md              # §2 기획문서 (빈 템플릿, /기획에서 채움)
├── README.md              # 한국어 README
├── .gitignore             # 표준 .gitignore
├── intake/                # §0 Discovery 산출물 (/인테이크)
│   └── .gitkeep
├── contracts/             # §1 견적·계약·입금 트래커 (/견적, /입금확인)
│   └── .gitkeep
├── reports/               # 진행 보고서·검수 보고서 (/클라이언트보고, /전수검사)
│   └── .gitkeep
├── docs/                  # 디자인, 사양서, 회의록
│   ├── design/
│   ├── specs/
│   └── meetings/
├── uat/                   # §6 UAT 시나리오·결과 (/UAT)
│   └── .gitkeep
├── acceptance/            # §8 인수확인서·인계매뉴얼 (/인수)
│   └── .gitkeep
├── warranty/              # §8 하자보수 로그·모니터링 가이드 (/하자보수)
│   └── .gitkeep
├── change-requests/       # Side CR 견적·미니 사이클 (/CR)
│   └── .gitkeep
└── src/                   # 코드 (기술 스택 결정 후 채움)
```

**참고**: `intake/`, `contracts/`, `uat/`, `acceptance/`, `warranty/`, `change-requests/`는 워크플로 v2.0에서 신설된 디렉토리입니다. 각 단계 스킬이 이 디렉토리에 산출물을 쓰니 미리 생성해둬 일관성 확보.

### Step 3 — 파일 채우기

**CLAUDE.md**: `~/.claude/templates/기획안.md` 복사 (첫 줄 `유형: 외주` 기본 포함 — 이 커맨드는 외주 부트스트랩이므로 외주가 맞다. 개인 사업이면 이 커맨드 대신 `/기획`으로 시작해 `유형: 개인`을 선언)
**README.md**: 다음 형식으로 작성

```markdown
# $ARGUMENTS

## 개요
(1단계 기획문서 작성 후 채움)

## 시작하기
(기술 스택 결정 후 채움)

## 산출물
- 기획안: ./CLAUDE.md
- 디자인: ./docs/design/
- 진행 보고서: ./reports/

## 클라이언트 정보
(필요 시 .gitignore에 추가된 별도 파일에)
```

**.gitignore**: 다음 포함
```
node_modules/
.env*
!.env.example
.DS_Store
*.log
dist/
build/
.next/
.vercel/
.railway/
client-info.md   # 클라이언트 민감 정보
```

### Step 4 — git 초기화
```bash
git init -b main
git add .
git commit -m "초기 셋업: 외주 프로젝트 디렉토리 구조"
```

### Step 5 — 1단계로 자동 진입
사용자에게 안내:
"부트스트랩 완료. 이제 클라이언트 요구사항을 받아 `/기획 [원문 요구사항]`으로 1단계를 시작하세요."

또는 사용자가 이미 요구사항을 알려줬다면 바로 `/기획` 실행.
