# 클라이언트 보고서 생성

보고 유형: $ARGUMENTS (없으면 git log + 현재 상태로 자동 판단)

## 절차

### Step 1 — 컨텍스트 수집
- 프로젝트 루트의 `CLAUDE.md` (기획문서) 읽기
- `git log --oneline --since="7 days ago"` 으로 최근 작업 파악
- 마지막 보고서 (`./reports/` 디렉토리에 있으면) 참조하여 중복 제거

### Step 2 — 보고 유형 결정
- 사용자가 명시했으면 그대로
- 자동 판단:
  - 마지막 보고 후 7일 지났음 → 주간 보고
  - 최근 커밋이 마일스톤 관련 → 마일스톤 보고
  - 최근 production 배포 있음 → 배포완료 보고

### Step 3 — 템플릿 적용
`~/.claude/templates/진행보고서.md` 템플릿 사용.
한국어, 클라이언트가 비전문가라고 가정하고 작성.

### Step 4 — 핵심 섹션 (모든 보고에 공통)

```markdown
# [프로젝트명] [보고유형] 보고서
작성일: YYYY-MM-DD

## 한 줄 요약
[전반적 진행 상황]

## 이번 기간 완료
- 기능 A
- 기능 B

## 진행 중
- 기능 C (예상 완료: ...)

## 다음 기간 계획
- ...

## 이슈 / 협의 필요 사항
- ... (없으면 "특이사항 없음")

## 일정 대비 진척률
- 전체 N% (마일스톤 X/Y 완료)

## 확인 부탁드립니다
- ... (클라이언트가 결정해줘야 할 것)
```

### Step 5 — 저장 및 출력
- `./reports/YYYY-MM-DD-보고서.md`로 저장
- 채팅에도 출력 (사용자가 바로 복사해서 클라이언트에게 전달 가능)
- PDF 변환이 필요하면 안내 (pandoc 명령 제시)
