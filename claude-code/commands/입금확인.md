# 입금 확인 (트래커 갱신)

당신은 외주 프로젝트의 **입금 트래커**를 갱신합니다.
**§11 게이트 ①** (착수금 입금 → §2 진입)이 작동하기 위한 기반 데이터입니다.

## 입력
$ARGUMENTS

## 절차

### Step 1 — 인자 파싱
- 첫 토큰: 단계명 — "착수금" / "중도금" / "잔금" / "CR-N" (N은 숫자)
- 둘째 토큰(선택): 입금일 — 없으면 오늘 날짜 (`date +%Y-%m-%d`)
- 누락이면 AskUserQuestion으로 묻기

### Step 2 — 트래커 파일 찾기
- 일반 단계(착수금/중도금/잔금): `./contracts/입금트래커.md`
- CR-N: `./change-requests/입금트래커-CR.md` 또는 CR 견적 파일 내 입금 섹션
- 파일이 없으면 "먼저 `/견적` 또는 `/CR`로 트래커 생성이 필요합니다" 안내 후 종료

### Step 3 — 입금일·상태 갱신
트래커 표의 해당 행을 수정:
- 입금일 칸: 인자에서 받은 날짜 (또는 오늘)
- 상태 칸: ⏳ → **✅**

예시 — 변경 전:
```markdown
| 착수금 | 30% | ₩300,000 | 계약일+7 | | ⏳ |
```

변경 후:
```markdown
| 착수금 | 30% | ₩300,000 | 계약일+7 | 2026-05-19 | ✅ |
```

### Step 4 — 게이트 통과 알림
- 착수금 ✅: "게이트 ① 통과. `/기획`으로 §2 진입 가능합니다."
- 중도금 ✅: "검수 통과 + 중도금 입금 확인. 후반 작업(UAT, 배포) 진행 가능."
- 잔금 ✅: "인수확인서 + 잔금 완료. 하자보수 모드(`/하자보수`)로 전환하세요."
- CR-N ✅: "CR-N 입금 확인. 미니 1→5 사이클 진입 가능."

### Step 5 — 로그
- 변경 사항 한 줄 요약을 채팅에 출력
- 트래커 파일 변경 commit (선택, 사용자 의도에 따라):
  ```bash
  git add ./contracts/입금트래커.md
  git commit -m "chore: 입금 확인 — {단계명} {입금일}"
  ```

## 절대 금지
- 클라이언트가 보내지 않은 입금을 임의로 ✅로 표시 (사용자에게 한 번 더 확인 권장)
- 트래커가 없는데 임의로 새 파일 생성
